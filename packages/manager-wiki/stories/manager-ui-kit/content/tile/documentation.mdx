<StorybookHeading label="Anatomy" level={2} />

1. **Card**: ODS Card; flexible container to display information in a structured manner
2. **Section**: Represents a section of the page with heading to identify the data in the card.
3. **Text** (h4): ODS Text; Text with preset `heading-4` to identify the section with heading.
4. **Description List**: `dl`; To encloses a list of groups of terms and descriptions.
    1. **Description Term**: `dt`; Specifies a term in the description list. Usually followed by `dd` element.
    2. **Description Details**: `dd`; provides the description, definition, or value for the preceding term.
    3. **Divider**: ODS Divider; indicates the end of a combination of `dt` and `dd` elements.

<StorybookHeading label="Placement" level={2} />

The **Tile** component is typically placed in the Dashboard below the `Tabs`.

<StorybookHeading label="Behavior" level={2} />

**Interactions**

Actions

- Actions Menu is displayed next to `Tile.Item.Term` that has a list of grouped actions related to the term.
- `Tile.Item.Description` can be used to display links to guides/actions related to the preceding term in `Tile.Item.Term`.

<StorybookHeading label="Variation" level={2} />

**Description List**

Tile can be used in Description List form to display group of related information the key-value pair. Description List itself can have multiple variations.
    1. Single Term and Description
    2. Muliple Terms and Single Description
    3. Single Term and Multiple Descriptions
    4. Multiple terms and Multiple Descriptions


<StorybookHeading label="Accessibility" level={2} />

_Ensure the component meets accessibility standards.
Describe what should keyboard interaction do to the component
WAI Patterns can help you go though keyboard interaction :_ **[https://www.w3.org/WAI/ARIA/apg/patterns/](https://www.w3.org/WAI/ARIA/apg/patterns/)**

To meet accessibility standards:

- Ensure heading is set for the Section element.
- Every `dt` element is a child of `dl` or `div` that is a child of `dl` element.
- Every `dd` element is a child of `dl` or `div` that is a child of `dl` element. This element can be used after a `dt` or another `dd` element.
