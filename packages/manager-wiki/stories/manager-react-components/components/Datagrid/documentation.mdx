# Datagrid

## Overview

The `Datagrid` is table component used for displaying and interacting with data

It supports advanced features like:

- **Sorting**: Enable column sorting with custom logic.
- **Infinite Scrolling**: Load additional data on demand with a "Load More" button.
- **Pagination**: Load additional data on demand with a "Load More" button.

## Usage

- Can be use with API V6 and API V2
- `Pagination` mode is now deprecated
- `Cursor` navigation is recommended <a target="_blank" href="https://apiv2.pages.ovhcloud.tools/deep-dive/seamless/pagination/">link</a>

The TanStack Table (formerly React Table) is a lightweight, powerful, and highly customizable library for building data grids and tables in React applications. It provides a modular architecture and advanced features to handle complex table requirements efficiently.

- **Flexibility**: Customize every aspect of the table, from rendering to state management.
- **Performance**: Lightweight and optimized for large datasets.
- **Feature-Rich**: Sorting, filtering, pagination, and virtualization out of the box.

### Filters

#### How to use It

1 - In your columns definition, fill `type` and/or `comparator` attributes

- `type` is a `FilterTypeCategories` and will allow you to have enhanced filters (datepicker for date type, etc...)

- `comparator` is a `string` of `FilterCategories`. Use this option to define the set of comparators that can be applied on the data. By default, Datagrid determines the set of comparators that can be applied depending on the `type` field. **Only use this option if default comparators does not match your need.**

<Source
  code={`
    export const columns = [
        {
            id: 'label',
            cell: (item: Item) => {
                return <DataGridTextCell>{item.label}</DataGridTextCell>;
            },
            label: 'Label',
            type: FilterTypeCategories.String
        },
        {
            id: 'price',
            cell: (item: Item) => {
                return <DataGridTextCell>{item.price} €</DataGridTextCell>;
            },
            label: 'Price',
            comparator: FilterCategories.String,
        },
        {
            id: 'creationDate',
            cell: (item: Item) => {
                return <DataGridTextCell>{useFormattedDate(item.creationDate)}</DataGridTextCell>;
            },
            label: 'Creation date',
            type: FilterTypeCategories.Date
            comparator: FilterCategories.Date,
        },
    ];
    `}
  language="javascript"
/>

2 - In the datagrid component, pass `filters` object

<Source
  code={`
import { useResourcesV6, useResourcesIcebergV6 } from '@ovh-ux/manager-react-components';

const { filters } = useResourcesIcebergV6({ ... });
const { filters } = useResourcesV6({ ... });

<Datagrid
    {...}
    filters={filters}
/>
    `}

language="javascript"
/>

If you are using a custom hook

<Source
  code={`
import { useColumnFilters } from '@ovh-ux/manager-react-components';

const { filters, addFilter, removeFilter } = useColumnFilters();

<Datagrid
    {...}
    filters={{filters: filters, add: addFilter, remove: removeFilter}}
/>

### Filters with Tags

#### How to use It

1 - In your columns definition, use `type: Tags` attribute to use iam Tag filters for this column

- `type` is a `FilterTypeCategories`

<Source
  code={`
    export const columns = [
        {
            id: 'tags',
            cell: (item: Item) => {
                return <DataGridTextCell>{JSON.stringify(item.iam?.tags)}</DataGridTextCell>;
            },
            label: 'Tags',
            type: FilterTypeCategories.Tags
        }
    ];
  `}
  language="javascript"
/>

2 - Get only tags for corresponding resource

By default, the tag filter will fetch all the user existing tags but most of the time you would like to have only the tags for the resource displayed in the datagrid.
To be able to get only the tags associate to the resource, you can use `resourceType` property on Datagrid definition.

- `resourceType` can be found on APIv2 /iam/reference/resource/type.

<Source
  code={`
<Datagrid
    {...}
    resourceType="dedicatedServer"
/>
    `}

language="javascript"
/>

**Note:** Filters on Tags type of column is currently supported only on the primary resources.

### Row selection

#### How to use It

To activate default row selection, use `manageRowSelection` attribute on the Datagrid definition.

```tsx
const [rowSelection, setRowSelection] = useState({});

return
  <Datagrid
    ...
    manageRowSelection={{
      rowSelection,
      setRowSelection,
      onToggleAllRowsSelection, // Optional
      enableRowSelection // optional
    }}
  />;
```

A new column is automatically displayed at the 1st place of the datagrid with checkboxes to select each row but also checkbox in the header to select / unselect all the rows.

#### Disable the selection for some of the rows

`enableRowsSelection` is a function that can be use inside `manageRowSelection` attribute to tell the Datagrid which row should be selectable or not.
The row is passed in the parameters of the function so it is easy to use on parent side.

example:

```tsx
const [rowSelection, setRowSelection] = useState({});

const isSelectable = (item: Item) => item.price > 10;

return
  <Datagrid
    ...
    manageRowSelection={{
      rowSelection,
      setRowSelection,
      enableRowSelection: ({ original: item }) => isSelectable(item)
    }}
  />;
```

In that case, only items with price > 10 would be selectable and others would have disabled checkbox. When using "select all" checkbox, only available checkboxes is selected.

#### Overwrite the select all behavior

By default, the select all checkbox is selecting only the items that the datagrid is aware of.
Because in manager we can have manual pagination on front end side, we maybe want the "select all" to select items that are not send on the datagrid yet.
It is possible to select items that are known by front but not the datagrid by overwriting the action made on select all using `onToggleAllRowsSelection` function inside manageRowSelection attribute

```tsx
const [rowSelection, setRowSelection] = useState({});

const onToggleAllRowsSelection = (isAllRowSelected: boolean) => {
    const selectedItems: Record<number, boolean> = {};
    items.forEach((item, index) => {
        selectedItems[index] = !isAllRowSelected;
    });
    setRowSelection(selectedItems);
};

return
  <Datagrid
    ...
    manageRowSelection={{
      rowSelection,
      setRowSelection,
      enableRowSelection
    }}
  />;
```

In the case of API pagination, the front don't know the entire list of items before fetching the API so it is not possible to select all items in advance.
When using "select all" checkbox, only the displayed items will be selected. If the user click on load more, the newly fetched items won't be selected and "select all" checkbox will get the indeterminate state.

#### Overwrite the rowId for better row selection

By default, the row id is the index of the item within the datagrid. To be able to better know which row has been selected,
it is possible to overwrite the row id with `getRowId` attribute.

`getRowId` is a function that have row item passed as a parameter and should return the custom id as a string.

```tsx
const getRowId = (item) => item.label;

return
  <Datagrid
    ...
    getRowId={getRowId}
  />;
```

when using this attribute, the selected rows will be defined by the custom id to be easier to work with.

In the case of using `onToggleAllRowsSelection`, be careful to create the rowSelection state object using custom id and not index:

```tsx
const [rowSelection, setRowSelection] = useState({});

const getRowId = (item) => item.label;

const onToggleAllRowsSelection = (isAllRowSelected: boolean) => {
    const selectedItems: Record<string, boolean> = {};
    items.forEach(({label}) => {
        selectedItems[label] = !isAllRowSelected;
    });
    setRowSelection(selectedItems);
};

return
  <Datagrid
    ...
    manageRowSelection={{
      rowSelection,
      setRowSelection,
      enableRowSelection
    }}
    getRowId={getRowId}
  />;
```
