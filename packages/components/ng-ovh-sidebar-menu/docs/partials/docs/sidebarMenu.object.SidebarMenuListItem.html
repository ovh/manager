SidebarMenuListItem
object in module sidebarMenu

Description

Factory that describe an item into manager sidebar menu.

Dependencies

  • $q

Usage

SidebarMenuListItem(options);

Parameters

ParamTypeDetails
optionsObject

Options for creating a new SidebarMenuListItem.

PropertyTypeDetails
allowSearch
(optional)
Boolean

Flag telling if item allow search. Only available for item of level 1.

(default: false)

allowSubItems
(optional)
Boolean

Flag telling if item allow to have sub items. If true will display an arrow icon before title.

(default: false)

category
(optional)
String

Add a category to the item. This will be added as a new class to the menu item element.

(default: none)

icon
(optional)
String

Icon added before prefix and title. Example : ovh-font icon!

id
(optional)
NumberString

Unique id of the SidebarMenuListItem.

(default: A random Number)

level
(optional)
Number

Menu item level.

viewAllItemObject

Links to a page that manages all subitems.

viewMore
(optional)
Object

Custom object to implement your own pagination method.

viewMore.titleString

The displayed title of the viewMore button.

viewMore.enabledBoolean

Should the viewMore button be displayed?

viewMore.loadingBoolean

Should a loading spinner be displayed near the viewMore button?

viewMore.actionFunction

Custom pagination callback to be called when viewMore button is pressed. If this callback returns a promise, the scrollbar will scroll automatically to the bottom after the promise is resolved (when your paginated items have been added).

loadOnState
(optional)
String

State that will automatically load the menu item. For this to work, states MUST be declared as parent/child (example of state name : parent.child.subchild).

loadOnStateParams
(optional)
Object

StateParams that will that defines the state that will automatically load the menu item. Ignored if no loadOnState option.

(default: {})

onLoad
(optional)
Function

Function called to load sub items. This function MUST return a promise.

parentId
(optional)
NumberString

Unique id of the parent SidebarMenuListItem.

(default: null)

prefix
(optional)
String

Prefix added befor item title.

state
(optional)
String

State name where to redirect when clicking on item. Will be ignored if both options (state and url) are setted.

stateParams
(optional)
Object

State params to add when switching state. Ignored if no state option.

(default: {})

status
(optional)
String

Add a status to the item. This will be added as a new class to the menu item element.

target
(optional)
String

Target attribute value that will be added to item link. Ignored if no url option.

(default: _blank)

titleString

Title of the the menu item. This is what will be displayed (cropped if too long and the entire title will be in the title attribute).

errorString

Error message that will be displayed if loding promise is rejected.

(default: none)

url
(optional)
String

Url to redirect when item link is clicked. Will be ignored if both options (url and state) are setted.

infiniteScrollBoolean

Should infinite scroll be used for scrolling subItems?

searchableBoolean

Flag telling if item is searchable, true by default, must be set to false to hide it from search.

(default: true)

  • appendPendingListItems()

    Append the pending list of items to the sub items list. If the list of pending items is large it can be better to call SidebarMenuListItem#appendPendingListItemsAsync in order to not freeze the brower by flooding the DOM with subItems directives.

    Returns
    SidebarMenuListItem

    Current instance of menu item.

  • appendPendingListItemsAsync()

    Append asynchronously the pending list of items to the sub items list. It allows to not freeze the browser when the list of pending items is large. Pending items are added in chunks of 5 sequentially by calling timeouts multiple times.

    Returns
    SidebarMenuListItem

    Current instance of menu item.

  • filterSubItems()

    Search for string "search" in item's searchKey and perform the search recursively in all subItems. Items not matching the "search" will be removed from the dom.

Example

angular.module("myManagerApp").controller("MyTestCtrl", function ($q, SidebarMenuListItem) {
    var myMenuItem = new SidebarMenuListItem({
        title : "My beautiful title",
        state : "manager.my.state"
        stateParams : {
            foo : "test",
            test : "foo"
        },
        icon : "dots",
        allowSubItems : true,
        onLoad : function () {
            return $q.when([]);
        }
    });
});
